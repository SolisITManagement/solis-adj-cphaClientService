DETAILED FINDINGS

17. Software systems for application hosting
---------------------------------------------
Verdict: NEEDS SPECIFICS

The answer is directionally correct but vague. Based on the actual stack:

  - Careverse (Client): Angular, TypeScript
  - Backend Microservices: Spring Boot 3.2.5, Java 21, Apache Maven
  - Compute: Google Cloud Run (serverless containers, NOT GKE)
  - Database: Google Cloud Firestore (NoSQL, NOT Cloud SQL/PostgreSQL)
  - Messaging: Google Cloud Pub/Sub (push subscriptions)
  - Container Registry: GCP Artifact Registry
  - Build: Google Cloud Build
  - Web Server: Embedded Apache Tomcat (within Spring Boot)
  - Monitoring: Cloud Logging (stdout), but no custom dashboards or alerting yet

Recommendation: Name Cloud Run, Firestore, Pub/Sub, and Artifact Registry
explicitly rather than keeping it generic.


18. Network infrastructure and capabilities
--------------------------------------------
Verdict: ASPIRATIONAL (not current state)

The answer mentions VPC, managed firewalls, load balancing, and DDoS
protection. Based on the codebase:

  - There is NO VPC configured -- Cloud Run services use default settings
  - There is NO Cloud Armor/WAF configured
  - There is NO explicit load balancer -- Cloud Run provides built-in
    request routing but no Global LB
  - DDoS protection is implicit via Cloud Run (accurate)

Recommendation: This answer describes the planned/target state (Phase 2
of the security plan), not the current state. If the questionnaire asks
about current state, qualify with "will be" language.
