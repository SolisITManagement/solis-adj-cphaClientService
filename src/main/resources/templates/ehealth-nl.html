eHealth — NL Pharmacy Network

Newfoundland & Labrador Pharmacy Network (NLPN) via HIAL — CeRx V01R04.3

Annotation / Patient Notes
Outbound POS → HIAL annotation requests
Add Patient Note (COMT_IN300001CA)
MandatoryCeRx
Add a free-text or coded annotation to a patient's NLPN record (e.g. patient preference, clinical observation).
Deprecate Patient Note (COMT_IN300101CA)
MandatoryCeRx
Mark an existing patient note as no longer applicable (correction of an erroneously entered note).
Patient Note Query (COMT_IN300201CA)
MandatoryCeRx
Retrieve notes recorded against a patient, optionally filtered by note type and/or date range.
Add Note To Record (COMT_IN301001CA)
MandatoryCeRx
Add an annotation against a specific record (prescription, dispense, device prescription, etc.) rather than the patient globally.
Retract Action
Outbound POS → HIAL nullification of a previously recorded ControlAct
Retract Action (COMT_IN600001CA)
MandatoryCeRx
Nullify a previously recorded event (e.g. an erroneously submitted dispense or annotation) by referencing its original event id.
Issue Management / DUR Override
Outbound POS → HIAL record of a pharmacist's management of a detected clinical issue
Issue Mgmt / DUR Override (PORX_IN050001CA)
MandatoryCeRx
Record how a detected DUR issue (interaction, allergy, duplicate therapy, etc.) was managed — typically an override with clinical justification.
Refusal to Dispense
Outbound POS → HIAL notification that a prescription was refused
Refusal to Dispense (PORX_IN090001CA)
MandatoryCeRx
Record that the pharmacist declined to dispense a prescription (e.g. documented allergy, unsafe therapy), with a coded reason.
Dispense Responsibility Transfer
Outbound POS → HIAL hand-off of dispensing responsibility to another pharmacist/facility
Dispense Responsibility Transfer (PORX_IN110001CA)
MandatoryCeRx
Transfer clinical/dispensing responsibility for a prior dispense event to a different provider or service delivery location.
Prescription Create — Medication + Device
Outbound POS → HIAL new prescription activation
Activate Prescription (PORX_IN010380CA)
MandatoryCeRx
Activate a new medication prescription in the patient's NLPN record — DIN, quantity, directions, refills and prescriber.
Activate Device Prescription (PORX_IN010300CA)
MandatoryCeRx
Activate a new device (e.g. glucose meter, spacer) prescription against the patient's NLPN record.
Prescription Suspend / Resume / Abort
Outbound POS → HIAL lifecycle changes to an existing prescription
Suspend Prescription (PORX_IN010440CA)
MandatoryCeRx
Temporarily suspend an active prescription (e.g. patient hospitalised, therapy paused) with a coded reason.
Resume Prescription (PORX_IN010520CA)
MandatoryCeRx
Resume a previously suspended prescription (e.g. patient discharged from hospital) with a coded reason.
Abort Dispense Authorization (PORX_IN010560CA)
MandatoryCeRx
Cancel the remaining dispense authorisation on a prescription (patient no longer needs further dispenses).
Abort Prescription (PORX_IN010840CA)
MandatoryCeRx
Abort an active prescription entirely (e.g. therapy changed, prescribed in error) with a coded reason.
Pre-determination / DUR Check Results
Inbound HIAL → POS — driven by the {@code simulate} field on Activate Prescription / Activate Device Prescription
Pre-determination OK (PORX_IN010640CA)
CeRx
Clinical pre-determination approved (inbound only). Trigger from Activate Prescription with Prescription.simulate = "predetermination-ok".
Pre-determination Not OK (PORX_IN010630CA)
CeRx
Clinical pre-determination raised DUR issue(s) (inbound only). Trigger from Activate Prescription (or Activate Device Prescription) with Prescription.simulate = "predetermination-not-ok".

Polling, Broadcast & Password (Section 23)
Send Poll Request (MCCI_IN100001CA)
MandatoryShared V02R02.0
Pull next queued message from the HIAL inbox. Default response is MCCI_IN100003CA (empty queue) or a queued unsolicited inbound (PRPA_IN101002/004/999, NLPN_IN100110) when simulate="next".
Poll & Fetch Next (MCCI_IN100004CA)
MandatoryShared V02R02.0
Acknowledge the previous polled message and immediately request the next. Default response is MCCI_IN100003CA.
Send Poll Exception Ack (MCCI_IN100005CA)
MandatoryShared V02R02.0
Notify HIAL that a polled message could not be consumed. Default response is generic MCCI_IN000002CA ack.
Query Broadcast Topics (NLPN_IN100120CA)
MandatoryNL-Defined V02R02.0
List the broadcast topics this user is subscribed to. Default response is NLPN_IN100130CA.
Subscribe Broadcast Topic (NLPN_IN100140CA)
MandatoryNL-Defined V02R02.0
Subscribe to a broadcast topic so future NLPN_IN100110CA notifications appear in the polling queue. Default response is NLPN_IN100150CA.
Update Password Request (NLPN_IN100200CA)
MandatoryNL-Defined V02R02.0
Change the POS user's NLPN account password. Default response is NLPN_IN100210CA.
